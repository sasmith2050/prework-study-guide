HTML

  • HTML is one of the three cornerstone languages used in web development
  • HTML is used to structure a page via a makeup of element which work together to provide content
  • common types of elements:
    • Text elements
    • Image elements
    • Header elements
    • List elements
  • Common element types used to divide and describe a page:
    • Meta-related elements
      • Metadata: data about our webpage that the browser uses to effectively serve our webpage to users.
    • Content division elements
    • Semantic elements
  • The head element contains information anout the webpage.
    • The title element defines the title for a webpage. Is required and extremely important for SEO
      • Search Engine Optimization(SEO): the process of imporving the ranking of a website in seach engine such as Goggle or Bing.
    • the style element defines the CSS styles associated with a website.
    • the base element defines the base URL for a webpage
    • the link element connects an external resource to the HTML document
    • the meta element defines metadata such as the character set, description, keywords, author, and viewpoint.
    • the script element embeds client-side scripts in the HTML document
  • The body element represents the visible content shown to the user.
    • Child Element: elements that are nested inside other elements
    • Parent Element: the element that holds all the child elements
    • Semantic HTML Elements: the name of the element describes its content and the role it plays on a webpage
    • Section elements are semantic elements that define a section in a document that contains thematic contect
  • HTML Commands:
    • head: element contains information that helps the browser to render the page correctly. Elements within are not visible to user.
    • !DocType: this declaration tells the browser what documents type is should expect when it is opened.
    • html: opening tag, which represents the root, or start, of our HTML document
    • lang=(insert lang): tells the browser the language for the webpage
    • title: element defines the title for a webpage
    • style: element defines the CCS styles associated with a webpage
    • base: element defines the base URL for a webpage
    • link: element connects an external resource to the HTML document
    • meta: element defines metadata such as the character set, description, keywords, author, and viewpoint
    • charset: character set, as UTF-8, which allows us to use most characters and symbols on the page
    • script: element embeds client-side scripts in the HTML document
    • body: element contains the content that is shown to the user that they can interact with
    • div: to create a division in the page
    • span: to create an inline division
    • h1, h2, h3, h4, h5, h6: elements represent the level of heading a given text block represents
    • p: element represents a paragraph or block of text
    • ul,ol,li: elements represent unordered lists, ordered lists, and list items
    • img: element contains information about images that are displayed on the webpage
    • src: attribute is the most important because it defines the location of the image
    • alt: attribute contains a text string that describes the appearance and functionality of the image
    • br: element creates a line of empty space, or a line break, between two blocks of content
    • a: element (which stands for anchor) creates links to the same webpage or other webpages
    • href: attribute points to the URL for the link
    • footer: appears at the bottom of the page and usually contains the author, copyright, contact, sitemap, and navigation
    • section: elements are semantic elements that define a section in a document that contains thematic content

CSS

  • Cascading Style Sheets(CSS): language used to describe presentation of website that's created with HTML and how elements on webpage is rendered
  • 3 ways to style a webpage using CSS:
    • Inline CSS: a developer syles an element directly in the HTML file
    • Internal CSS: contains CSS rules for the webpage in the head section of the HTML file
    • External CSS: allows developers to keep all CSS rules in a seperate file, which makes design changes easier
      • This is the most common and preferred style
  • seperation of concern: seperating code into different files based on use
  • Declaration: contain two important components: (1) the CSS property we want to apply and (2) the value of the property
  • Do Not Repeat Yourself(DRY): a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained
  • Class Attribution: allow us share a CCS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • CSS Commands:
    • Margin: indicates how much space around the outside of an element
    • padding: adds space around the content inside an element
    • *: add when we want to apply a rule to all the elements visible on the page
    • display: assigned with the block value
    • block: assigns each image to start a new line
    • ul: delares the font and the padding on the left of the unordered list
    • link: creates the connection to the external file
    • rel: specifies the relationship between the current document and the linked document resource
    • href: specifies the location(URL) of the external resource
    • card: class selector

Git

  • Git: software on your computer that tracks changes that you make to your code
  • GitHub: a cloud-based service that allows developers to store, manage, and collaborate on code
  • Branch: Workspace where you can make changes that don't affect the live switches
    • Main Branch:
    • Side Branch:
  • Markdown(.md>): the way content files are formatted in GitHub
  • commit changes: saving a new version of a file you're working on
  • Pull request(PR): Asking for changes on a branch to merge into another branch
  • Merge: When PR is approved, changes are "pulled in" or mereged into main branch
  • Issue: Way to report new tasks or content fixes
    • Track progress on progress board
    • Open issue for new tasks and close once done
  • Blocker: you're stuck on a particular problem that is preventing you from moving forward
  • Bottleneck: a point in our development process that is slowing down the rest of production
  • all git code starts with git in the front
  • Git Commmands:
    • git config --global user.name "(firstname lastname)": set a name that is identifiable for credit when review version history
    • git config --global user.email :"(valid-email)": ser an email address that will be associated with each history marker
    • git config --global color.ui auto: ser automatic command line coloring for Git for easy reviewing
    • git init: initialize an existing directory as a Git repository
    • git clone: retrieve an entire repository from a hosted location via URL
    • git status: show modified files in working directory, staged for your next commit
    • git add (file): add a file as it looks now to your next commit(stage)
    • git reset (file): unstage a gile whole retaining the changes in working history
    • git diff: diff of what is changed but not staged
    • git diff --staged: diff of what is staged but not yet committed
    • git commit -m "(descriptive message)": commit your staged content as a new commot snapshot
    • git branch: list your branch. a* will appear next to the currently active branch
    • git branch (branch name): create a new branch at the current commit
    • git checkout: switch to another branch and check it out into your working directory
    • git merge (branch): merge the specified branch's history inot the current one
    • git log: show all commits in the current banch's history
    • git log: show the commit history for the currently active branch
    • git log branchB..branchA: show the commits on branchA that are not on branchB
    • git log ---follow (file): show the commits that changed file, even across renames
    • git diff branchB ...branchA: show the diff of what s in branchA that is not in branchB
    • git show (SHA): show any object in Git in human-readable format
    • git rm (file): delete the file from project and stage the removal for commit
    • git mv (existing-path) (new path): change an existing file path and stage the move
    • git log --stat -m: show all commit logs with indication fo any paths that moved
    • logs/ .notes pattern/: Save a file with desired patterns as .gitignore with either direct string matches or wildcard globs
    • git config --global core.excludesfile (file): system wide ignore pattern for all local repositories
    • git remote ass (alias) (url): add a git URL as an alias
    • git fetch (alias): fetch down all the branches form that Git remote
    • git merge (alias)/(branch): merge a remote branch into your current branch to bring it up to date
    • git push (alias) (branch): Transmit local branch commits to the remote repository branch
    • git pull: fetch and merge any commits from the traching remote branch
    • git rebase (branch): apply any commits of curreny branch ahead of specidied one
    • git reset --hard (commit): clear staging area, rewrite working tree from specified commit
    • git stash: save modified and staged changes
    • git stash list: list stack-order of stashed file changes
    • sit stash pop: write working form top of stash stack
    • git stash drop: discard the changes from top of stash stack
